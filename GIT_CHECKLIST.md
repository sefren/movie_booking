## 🚀 Deployment Preparation Steps

### 1. Verify Your .gitignore

Your `.gitignore` should include:
```
# Dependencies
node_modules
backend/node_modules

# Build outputs
dist
dist-ssr
*.local

# Environment variables
.env
.env.local
.env.*.local
backend/.env

# Editor directories and files
.vscode/*
!.vscode/extensions.json
.idea
.DS_Store
```

### 2. Check What Will Be Committed

Run this to see what Git will track:
```bash
git status
```

To see what's being ignored:
```bash
git status --ignored
```

### 3. Use the Automated Scripts

**Quick Check:**
```bash
check-deploy.bat
```
This verifies:
- Node.js, npm, Git are installed
- All required files exist
- No sensitive files will be committed
- Git repository status

**Prepare & Commit:**
```bash
prepare-deploy.bat
```
This will:
- Initialize Git (if needed)
- Verify .gitignore
- Show what will be committed
- Commit your code safely
- Guide you to push to GitHub

## 🎯 Quick Git Commands Reference

### Initial Setup
```bash
# Initialize Git (if not done)
git init

# Add all files
git add .

# Commit
git commit -m "Initial commit"

# Add remote
git remote add origin https://github.com/username/repo.git

# Push to GitHub
git branch -M main
git push -u origin main
```

### Regular Updates
```bash
# Check status
git status

# Add changes
git add .

# Commit with message
git commit -m "Your update message"

# Push to GitHub
git push
```

### Checking What's Tracked
```bash
# See tracked files
git ls-files

# See untracked files
git ls-files --others --exclude-standard

# See ignored files
git status --ignored
```

### Fixing Mistakes
```bash
# Undo last commit (keep changes)
git reset --soft HEAD~1

# Remove file from staging
git reset HEAD filename

# Discard local changes
git checkout -- filename
```

## ⚠️ Common Mistakes to Avoid

### ❌ DON'T:
- Commit `.env` files with real secrets
- Commit `node_modules/` folders
- Commit `dist/` build folders
- Push without testing locally first
- Use weak passwords for production
- Share API keys publicly
- Commit IDE-specific files (`.idea/`, `.vscode/`)

### ✅ DO:
- Use `.env.example` for templates
- Test locally before deploying
- Use strong secrets in production
- Keep `.gitignore` updated
- Commit regularly with clear messages
- Use different API keys for dev/prod
- Document your environment variables
